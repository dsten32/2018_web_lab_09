Web Lab 09 landing page

By default when you run a Java Servlet application, IntelliJ will open your chosen browser to the 'root' of the web application. If you do not have a resource mapped to the <url-pattern> of / then you will get a 404 error as the Tomcat server does not know what to display for you. To avoid this error you can configure what file to display by default by specifying file names in the (optional) <welcome-file-list> tag in your web.xml file. If this is not specified, Tomcat will by default try to serve you a file named index.html or index.jsp.

To show that you can do this, try renaming this file (currently named web09.html) to something else of your choice. Follow up by adding to the <welcome-file-list> block a new <welcome-file> entry to match your chosen name. If you re-run your application you should still see this page.
